Voice & Telegram

Once connected, you can control the robot by voice or Telegram.

Voice API is running at http://localhost:8088 — open this on your phone to use the chat UI.
Telegram — run the bridge in a separate terminal:
python3 -u tools/telegram_bridge.py
Needs MBOT_TELEGRAM_TOKEN in your .env file.

Troubleshooting

BLE: "not found"
  • Wait for CyberPi to fully boot (shows home screen, ~15 seconds)
  • Make sure Bluetooth is enabled on your laptop
  • On Linux: sudo apt install libdbus-1-dev is needed for BLE
  • Try again — BLE scanning can take a few seconds
USB: "permission denied" or "port not found"
  • Use a DATA cable, not a charge-only cable
  • On Linux: sudo usermod -a -G dialout $USER then log out and back in
  • On macOS/Windows: install CH340 driver from Makeblock
  • Check port: ls /dev/ttyUSB* or ls /dev/tty.usbserial*
Robot not responding to commands
  • Check the log output above for errors
  • Make sure CyberPi is powered on (not just the mBot2 chassis)
  • Try power cycling the CyberPi
  • If using BLE, make sure no other app is connected to it
