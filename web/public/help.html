ü§ñ What is mBot2 RuVector?

RuVector transforms your mBot2 into an AI-powered companion with a simulated nervous system. Instead of following rigid scripts, your robot develops personality-driven behaviors that emerge from its internal state (tension, coherence, energy) and environmental stimuli.

Core Concepts

  • Homeostasis: The robot maintains internal balance through tension, coherence, and energy levels
  • Reflex Modes: Four behavioral states (Calm, Active, Spike, Protect) that change based on stimuli
  • Personality: 9 parameters that shape how the robot reacts and expresses itself
  • Emergent Behavior: Complex behaviors emerge from simple rules + personality + environment
üí° Think of it like this:

Traditional robots: "If sensor reads X, do Y"

RuVector mBot2: "I feel tense because something is close. My curious personality makes me approach cautiously rather than flee. My energy is high, so my movements are expressive."

üöÄ Getting Started

Step 1: Build the Firmware

Follow the Download Guide to build and flash RuVector firmware to your mBot2's CyberPi.

Step 2: Start the Dashboard

  1. Open a terminal in the web directory
  2. Run npm install (first time only)
  3. Run npm start
  4. Open http://localhost:3000 in your browser

Step 3: Connect Your Robot

The dashboard will automatically connect to your mBot2 when it's nearby. You'll see the status indicator turn green.

  • Charged mBot2 batteries

üé® ArtBot - Drawing Companion

Hardware Setup

  • Pen servo attachment (45¬∞ up, 90¬∞ down positions)
  • Drawing surface (A4 paper recommended)

How Mood Affects Drawing

ArtBot's drawing style changes based on its emotional state:

  • Calm (tension < 0.3): Smooth, flowing lines with gentle curves
  • Tense (tension 0.3-0.7): Jagged, energetic strokes with sharp angles
  • Stressed (tension > 0.7): Chaotic scribbles, erratic movements

Drawing Sessions

  1. Deploy ArtBot from the Projects page
  2. Set desired personality (try "Zen" for calm drawings, "Excitable" for energetic art)
  3. Place paper under the robot
  4. Activate drawing mode on CyberPi
  5. Watch the robot create personality-driven artwork!

  • Voice command ("your turn")

Emotional Responses

The robot reacts to game events:

  • Winning: Happy animation (green LEDs, victory sound)
  • Losing: Sad animation (blue LEDs, disappointed sound)
  • Thinking: Concentration indicator (yellow LEDs, hum sound)

üî® LEGOSorter - Organization Helper

Hardware Requirements

  • Gripper attachment for picking pieces
  • Carousel station system (8 bins recommended)
  • RGB sensor for color detection
  • Good lighting (avoid shadows)

Supported Colors

Standard (11): Red, Blue, Yellow, Green, White, Black, Orange, Lime, Magenta, Cyan, Gray

Rare (3): Pearl, Chrome, Glow-in-Dark

Sorting Process

  1. Calibration: Run servo calibration (¬±2¬∞ accuracy)
  2. Setup: Configure carousel stations for each color
  3. Detection: Robot scans play area in grid pattern
  4. Sorting: Pick-drop loop for each piece (max 3 retry attempts)
  5. Verification: Double-check piece color after drop
  6. Inventory: Track sorted pieces by color and count

Tips for Best Results

  • Use consistent lighting (natural daylight or bright white LED)
  • Clean LEGO pieces before sorting
  • Avoid mixing transparent and opaque pieces
  • Start with simple colors (red, blue, yellow) before advanced ones
  • Calibrate regularly for accuracy

üìö LearningLab - Education Kit

Educational Experiments

1. Startle Response Experiment

Learn how the nervous system reacts to sudden stimuli.

  • Observe baseline tension levels
  • Trigger startle with loud sound or close approach
  • Measure tension spike and recovery time
  • Try different personalities to compare responses

2. Approach/Avoid Behavior

Explore how curiosity and fear balance decision-making.

  • Place novel object near robot
  • Watch approach/retreat cycles
  • High curiosity = more approaches
  • High startle sensitivity = more retreats

3. Recovery Time Analysis

Measure how quickly tension returns to baseline.

  • Record tension before stress event
  • Trigger stress (obstacle, loud noise)
  • Time until tension returns to ¬±10% of baseline
  • Compare across personalities

4. Mode Transition Observation

Understand the four reflex modes and their triggers.

  • Start in Calm mode (distance environment)
  • Gradually approach to trigger Active ‚Üí Spike ‚Üí Protect
  • Note behavioral changes at each threshold
  • Withdraw and observe return to Calm

Using the Visualizer for Learning

  1. Open the Neural Visualizer
  2. Run an experiment with your robot
  3. Watch real-time meters respond to stimuli
  4. Take notes on relationships (e.g., "When tension rises, energy drops")
  5. Try different personalities and compare results

üîß Troubleshooting

Dashboard Issues

Can't connect to robot

  • Check robot is powered on and within Bluetooth range
  • Verify WebSocket server is running (check terminal output)
  • Try refreshing the browser page
  • Check firewall isn't blocking port 8081

Personality changes not applying

  • Ensure robot is connected (green status indicator)
  • Check server console for "Personality update" messages
  • Try clicking "Apply to Robot" again
  • Restart the robot if parameters seem stuck

Robot Behavior Issues

Robot not responding to environment

  • Check sensors are clean and unobstructed
  • Verify ultrasonic sensor is functioning (check visualizer)
  • Try increasing startle sensitivity in personality mixer
  • Ensure robot isn't stuck in Protect mode

Movements are jerky or uncoordinated

  • Check battery level (low battery causes erratic behavior)
  • Increase coherence baseline in personality mixer
  • Ensure surface is smooth and level
  • Check wheel encoders are clean

Hardware Issues

Pen servo not working (ArtBot)

  • Verify servo is connected to correct port
  • Run servo calibration from Projects page
  • Check servo angles (45¬∞ up, 90¬∞ down)
  • Test servo manually with test program

Color detection inaccurate (LEGOSorter)

  • Improve lighting (avoid shadows and colored light)
  • Calibrate RGB sensor for your environment
  • Clean sensor lens
  • Check LEGO pieces aren't dirty or faded
